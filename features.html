Aide mémoire Markdown

CodiMD, tout comme HackMD, permet un travail collaboratif de création de page web ; ils utilisent la syntaxe Markdown.

  • Écrire directement en .html est compliqué…
  • L’idée est de partir d’un texte brut.
  • De lui faire subir un minimum de modifications pour en augmenter le sens.
  • Un moteur de rendu créera un joli fichier .html à publier.
  • C’est la méthode la plus rapide pour créer du contenu à base essentiellement de texte.

Les bases du Markdown

L’architecture du document

Les parties sont structurées avec des titres précédés de 1 à 6 croisillons # suivant leur niveau importance. Cela donne du sens au texte, ce n’est pas que de la mise en forme ; cela permet :

  • d’obtenir une table des matières automatique ;
  • de pouvoir définir un style commun à tous les titres de même niveau.

Regarder le document source de cette page !

Les listes à puces

Listes numérotées

On commence par le numéro de son choix (souvent 1), on ajoute un point puis une espace. Ensuite la numérotation est automatique.

Si on ajoute ou retire ensuite des lignes, la numérotation respectera la progression en partant de la première.

  1. Je commence ici à 1, par exemple.
  2. Peu importe ensuite les numéros…
  3. J’avais écrit trois belles lignes, elles sont effacées.
  4. Celle-ci était après la 5.

Listes non numérotées

  • On utilise, au clavier, les puces - ou * ou +, puis une espace avant chaque item.
  • On peut faire une liste à l’intérieur d’une liste, en la décalant avec une tabulation.
  • Par exemple pour dire
    • le signe plus : +
    • le signe moins : -
    • l’astérisque : *
      • et on pourrait
        • continuer
          • longtemps
            • l’imbrication…

L’emphase

Pour faire ressortir certains mots, on utilise de l’emphase forte ou bien de l’emphase faible.

  • L’emphase forte est souvent en gras.
  • L’emphase faible est souvent en italique.
  • On peut combiner les deux.

Elle doit s’utiliser avec justesse ; le moins possible.

Les tableaux

On peut faire des tableaux de plusieurs lignes et plusieurs colonnes en suivant le modèle :

Nom Note Commentaire
Alain 20 Bravo
Béatrice 10 Bon début
Charles 9 Manque de souffle
Louis 16 Tu vas perdre la tête

Pour les colonnes :

  • |---| signifie : comportement par défaut ;
  • |:---| signifie : justification à gauche ;
  • |---:| signifie : justification à droite ;
  • |:---:| signifie : justification centrée.

Les cases à cocher

On ajoute un [ ] ou un [x] à la suite d’une puce, avec de l’espace. On obtient des cases qui sont (dé-)cochées.

  • Boulot
    • Corriger les copies
    • Préparer les cours
    • Présenter sa discipline
  • Maison
    • Isoler le toit
    • Construire une cheminée
    • Refaire la sdb

Inclure du code Python

Un script :









def factorielle(n):
    """
    Renvoie le produit 1 × 2 × 3 × ... × n
    """
    if n == 0:
        return 1
    else:
        return n * factorielle(n - 1)

Une utilisation console :

>>> factorielle(5)
120
